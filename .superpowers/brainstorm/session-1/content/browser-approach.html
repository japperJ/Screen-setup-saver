How should browser URLs be captured?

This is the trickiest part — each approach has real trade-offs

Chrome DevTools Protocol
ws://localhost:9222/json
→ "url": "https://github.com/..."

A — CDP (Recommended)

Connect to Chrome/Edge's built-in debug port. Gets all open tabs, exact URLs. Requires browser launched with --remote-debugging-port flag — a one-time setup shortcut.

Pros

  • 100% accurate
  • All tabs, all windows
  • Real-time

Cons

  • Needs debug shortcut set up once
AppData\Local\Google\Chrome\
User Data\Default\Sessions\
⚠ locked while Chrome is open

B — Session Files

Read Chrome/Edge's session/history SQLite files directly from disk. No setup needed, but files are locked while the browser is open and the format changes between versions.

Pros

  • No browser config needed

Cons

  • Files locked while open
  • Brittle format
UI Automation API
FindElement(AddressBar)
→ reads URL from address bar

C — UI Automation

Use Windows Accessibility (pywinauto/uiautomation) to read the URL from the browser's address bar. Gets only the active tab per window.

Pros

  • No setup
  • Works with any browser

Cons

  • Active tab only
  • Fragile to UI changes
